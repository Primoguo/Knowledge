## 简介
本文件系统面向 Knowledge 应用，提供多格式文档的文本提取、网页内容清洗、OCR 文字识别、ZIP 解压以及 Office 文档解析能力。其目标是让用户通过统一的入口导入 PDF、Markdown、纯文本、EPUB、Office 文档（docx/xlsx/pptx）和网页链接，并自动转换为可阅读的纯文本，以便后续进行语音合成、摘要生成等上层功能。

**更新** 最新版本增强了HTML清理算法，特别针对现代单页应用(SPA)网站的复杂HTML结构进行了优化，能够更有效地处理动态生成的内容和残留标签。

#### 网页内容清洗算法
目标：从 HTML 中提取可读正文，过滤导航、广告、脚本、样式等无关区域，并对结果进行噪声行过滤与空行合并。

**更新** 核心清洗流程现在包含增强的HTML残留清理步骤，专门处理现代Web应用中常见的HTML标签泄漏问题。

核心步骤：
- 请求网页并正确解码编码
- 优先从 meta 标签（如 og:title、twitter:title）或 <title> 获取标题
- 正文定位策略：
  - 移除 head/script/style/noscript/nav/footer/header/aside 等标签
  - 优先匹配 article 标签
  - 其次匹配常见正文容器 class/id（正则集合）
  - 回退到 body 或最终返回已清洗的 HTML
- 使用 NSAttributedString 将 HTML 转为纯文本，若失败则回退到手动去标签
- **新增** 调用 `stripHTMLResidue` 方法进行深度HTML残留清理
- 后处理：过滤噪声行、URL、数字分隔线，合并多余空行

```mermaid

Fetch --> Title["提取标题(og/twitter/title/host)"]
Title --> CleanPre["去除无关标签与注释"]
CleanPre --> Locate{"定位正文区域"}
Locate --> |article| UseArticle["使用article内容"]
Locate --> |class/id| UseSelector["匹配常见正文容器"]

Residue --> Post["清洗文本(噪声/URL/空行)"]
Post --> End(["返回(title, cleanedText)"])
```

**更新** 新增了HTML残留清理步骤，专门处理SPA页面产生的标签碎片。

#### 增强的HTML残留清理功能
**新增** `stripHTMLResidue` 方法专门处理单页应用(SPA)页面产生的HTML标签残留问题，包含6个清洗阶段：

1. **完整标签去除**：移除所有完整的HTML标签及其属性
2. **属性碎片清理**：处理未闭合标签泄漏的class、id、data-*等属性
3. **HTML实体转换**：转换各种HTML实体字符为对应的Unicode字符
4. **不完整标签片段处理**：清理孤立的标签开头或结尾片段
5. **空白字符规范化**：合并多余的空白字符
6. **特殊字符清理**：处理数字实体和其他特殊编码

```mermaid
flowchart TD
Input["输入文本"] --> Stage1["阶段1: 完整标签去除<br/><[^>]+> → 空字符串"]
Stage1 --> Stage2["阶段2: 属性碎片清理<br/>class/id/data-*属性 → 空字符串"]
Stage2 --> Stage3["阶段3: HTML实体转换<br/>&nbsp;→ 空格, &amp; → &, 等"]
Stage3 --> Stage4["阶段4: 不完整标签处理<br/></?\\w+\\s* → 空字符串"]
Stage4 --> Stage5["阶段5: 空白字符规范化<br/>[ \\t]{2,} → 空格"]
Stage5 --> Output["输出清理后的文本"]
```

**更新** 这个新的清理方法显著提高了现代Web应用内容的提取质量。

#### EPUB 与 ZIP 解压
流程：
- 在临时目录解压 EPUB（ZIP 格式）
- 读取 META-INF/container.xml 找到 content.opf
- 解析 spine 中的 itemref，映射 manifest 中的 href，得到内容文件列表
- 按序读取每个内容文件，用 NSAttributedString 解析 HTML 并拼接文本
- ZIP 解压采用纯 Swift 实现，支持 Store 与 Deflate 压缩

## 性能考虑
- PDF 直接提取优先，仅在无文本时触发 OCR，避免不必要的图像渲染与识别开销
- 网页正文定位尽量早剔除无关标签，减少后续解析与清洗成本
- **新增** HTML残留清理采用高效的正则表达式处理，避免过度消耗CPU资源
- EPUB 解压使用临时目录并在完成后清理，避免长期占用磁盘
- Office 文档解析依赖系统框架，注意大文档可能带来的内存峰值
- 建议在批量导入场景中对网络请求与 OCR 任务进行并发控制与超时保护（当前实现未显式配置）

**更新** 新的HTML清理算法经过优化，在处理现代Web应用内容时能够在保持高质量的同时维持良好的性能表现。

## 故障排查指南
- 不支持的文件类型：检查扩展名是否在 DocumentType 与 extractText 分支中覆盖
- 文件未找到或无法访问：确认文件路径有效且具备安全访问权限（尤其是共享扩展或第三方应用传入的 URL）
- 提取失败：
  - PDF：若 OCR 失败，可能是图片质量过低或扫描模糊
  - 网页：检查 URL 有效性、编码是否正确、正文是否被反爬或动态加载
  - **新增** SPA页面内容：如果提取结果仍包含HTML标签，检查网站的JavaScript渲染情况
  - EPUB：确认 container.xml 与 opf 结构符合规范
  - Office：确认文件格式为 RTF 兼容，或系统能解析该 Office 变体
- 空内容：确保文件确实包含可提取文本，或在导入前进行预检

**更新** 对于现代Web应用，如果遇到HTML残留问题，建议检查目标网站是否为单页应用，可能需要额外的JavaScript执行环境来获取完整内容。
