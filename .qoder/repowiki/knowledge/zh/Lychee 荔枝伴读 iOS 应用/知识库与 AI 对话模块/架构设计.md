采用 SwiftUI + SwiftData 的 MVVM 分层：
- Models：`KnowledgeEntry` 为 `@Model` 实体，通过 `sourceRaw`/`categoryRaw` 字符串字段配合计算属性暴露 `KnowledgeSource`、`KnowledgeCategory` 枚举；`KnowledgeChat` 以 JSON 编码的 `messagesData` 存储对应条目的对话历史。
- Services：`KnowledgeService` 是单例，封装系统提示词模板、调用 `ServerAPIClient.shared.requestCompanion` 发起通义千问请求，并维护最多 20 条的内存对话上下文，同时通过 `loadConversation/saveConversation/clearConversation` 与 SwiftData 同步。
- Views：`KnowledgeListView` 使用 `@Query(sort: \updatedAt, order: .reverse)` 驱动列表，支持按 `KnowledgeCategory` 横向筛选与全文搜索；`KnowledgeDetailView` 在详情页内嵌聊天区，通过 `onAppear` 加载历史并在发送消息时异步调用服务后回写持久层。
- App：`KnowledgeApp` 作为 `@main` 入口，注册 `Document`、`CompanionChat`、`KnowledgeEntry`、`KnowledgeChat` 四张表到共享 `ModelContainer`，并通过 `.modelContainer()` 注入给子树。
- 导航：`ContentView` 是应用级 TabView，将 `KnowledgeListView` 作为“知识库”标签页嵌入，与书库、播放器、设置并列。
依赖方向：Views → Service → ServerAPIClient；Models 仅依赖 Foundation/SwiftData，无上层耦合。
