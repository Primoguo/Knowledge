- 所有数据变更都通过 `@Environment(\.modelContext)` 获取上下文，并使用 `insert`/`delete` + `try? modelContext.save()` 完成持久化。
- 异步网络或 I/O 操作（网页抓取、文件复制）统一用 `Task { await ... }` 启动，并在成功/失败分支中通过 `await MainActor.run` 回到主线程更新 UI 状态。
- 用户反馈采用 `@State` 布尔标志 + `.alert(... isPresented:)` 模式，错误信息集中存入 `alertMsg` 后统一呈现。
- 文件类型相关展示（图标名、颜色、显示名）全部委托给 `document.fileType` 的属性，视图层不做扩展名判断。
